# 0.1.0 — 2026-06-13

## Added

- **`ourairports_search_airports`** — full-text + faceted airport search (name/municipality/keywords, country/region/type), ranked, closed-excluded by default, truncation disclosure.
- **`ourairports_get_airport`** — full airport record by any code (IATA/ICAO/GPS/local/ident) with runways and frequencies inline; echoes the complete code set and a `resolvedVia`/`resolutionNote` (ambiguity warning for shared national codes); `unknown_code` error for no match.
- **`ourairports_find_airports`** — airports within a radius of a coordinate, ranked nearest-first by great-circle distance with bearing.
- **`ourairports_find_navaids`** — navaids near a coordinate or serving an airport (two modes, exactly one required); frequencies surfaced in kHz and MHz.
- **`ourairports_list_countries`** — countries with ISO codes and airport counts, optional continent filter and nested regions.
- **`airport://{code}` resource** — stable-URI twin of `ourairports_get_airport`.
- **`airport-data` service** — loads the six OurAirports CSVs into in-memory indices at `setup()` (id maps, priority-ordered unified code index, airport-ref joins, ident-keyed navaids, flat `Float64Array` coordinates, country/region maps, tokenized text search). CSVs parsed by header name, not column position.
- **`scripts/build-data.ts`** — fetches and bundles the six CSVs from the daily OurAirports GitHub mirror into `data/` at build time; the server never touches the network at runtime.
- **Config** — `OURAIRPORTS_DATA_DIR` (override the bundled data path) and `OURAIRPORTS_DEFAULT_SEARCH_LIMIT`. No API key, no secrets, single-tenant; all data is public-domain and bundled in the artifact.
