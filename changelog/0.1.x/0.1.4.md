# 0.1.4 — 2026-06-30

## Changed

- **Framework skills re-synced** with the framework bump — the `api-utils` parsing reference (js-yaml v5 / `YAML11_SCHEMA`) and the `polish-docs-meta` README reference (drops the `Contributing` section).

### Dependency bumps

- `@cyanheads/mcp-ts-core` ^0.10.9 → ^0.10.10
- `@biomejs/biome` 2.5.0 → 2.5.1
- `@types/node` 26.0.0 → 26.0.1

## Fixed

- **`hasTokenPrefix` forward-only matching** — a query token matches an indexed token it is a prefix of (`intern` → `international`), but no longer matches a shorter indexed token it merely starts with, so a gibberish query like `bzzqxw` no longer false-matches via a bare single-letter token. ([#1](https://github.com/cyanheads/ourairports-mcp-server/issues/1))
- **Ident-priority code index** — the unified code index is built after all rows load, in global priority passes (`ident`, then `icao_code` / `iata_code` / `gps_code` / `local_code`), so a globally-unique `ident` is never shadowed by an earlier CSV row's national code. ([#2](https://github.com/cyanheads/ourairports-mcp-server/issues/2))
- **No-searchable-terms notice** — a non-blank `query` that tokenizes to nothing (only stopwords or punctuation) now returns zero matches with a `noSearchableTerms` flag, and `ourairports_search_airports` surfaces a distinct notice instead of the generic empty-result guidance; an omitted or blank query still browses by facets. ([#3](https://github.com/cyanheads/ourairports-mcp-server/issues/3))
- **`find_airports` / `find_navaids` default limit** — `find_airports` now honors `OURAIRPORTS_DEFAULT_SEARCH_LIMIT` (default 20) when `limit` is omitted instead of a hardcoded 10, and both tools clamp the config-derived default to their own max of 50 (the config ceiling is 100). ([#4](https://github.com/cyanheads/ourairports-mcp-server/issues/4))

## Security

- **Transitive `js-yaml` DoS advisory cleared** — a lockfile re-resolve moves the transitive `js-yaml` to 3.15.0, clearing [GHSA-h67p-54hq-rp68](https://github.com/advisories/GHSA-h67p-54hq-rp68) (CVE-2026-53550, medium — quadratic-complexity DoS in YAML merge-key handling). Transitive build-tooling dependency, not exercised by the server's runtime, which parses CSV.
